Peer discovery (emit)

The emit module handles UDP discovery for 10kgp. It opens one UDP socket with SO_BROADCAST and SO_REUSEADDR, bound to port 9999 (fixed in main.c for the CLI).

Three transmission paths

Every pulse sends the local peer ID on all of the following:

  1. IPv4 broadcast targets — Subnet-directed broadcasts plus limited broadcast (255.255.255.255), collected from interfaces (emit_bcast.c).
  2. Multicast — Group 239.255.43.210 (RFC 2365 administratively scoped), same port; the socket joins the group per interface (emit_mcast.c).
  3. LAN unicast sweep — For each local IPv4 subnet that looks “LAN-sized” (mask implies at most 254 hosts, typically /24), directed UDP to each address .1–.254 except local IPs (emit_lan_scan.c). This helps when access points block wireless broadcast or multicast between clients.

Pulses are sent every PULSE_INTERVAL_SEC (5 seconds) and can be triggered manually with /refresh.

Packet format

The UDP payload is the sender’s peer ID as a null-terminated ASCII string (length limited by EMIT_PEER_ID_LEN and the send buffer).

Receiving and the main loop

  • emit_listen(callback, user_data, timeout_ms) — Blocks on select until the socket is readable or the timeout elapses; then drains queued datagrams with recvfrom(..., MSG_DONTWAIT) until the queue is empty.
  • emit_recv_drain(callback, user_data) — Drains queued datagrams only (non-blocking). The 10kgp main loop calls this after every successful select (whether stdin or the socket woke up), because on some hosts the UDP socket can have queued packets while select omits it from the readable set when stdin is monitored in the same fd set.

Diagnostics

Set environment variable 10KGDP_TRACE=1 to print pulse and receive lines to stderr (helpful on restrictive networks).

10KGDP_TRACE=1 ./build/10kgp

C API summary

FunctionRole
emit_init(uint16_t port)Create socket, bind, enlarge buffers, join multicast
emit_set_peer_id(const char *id)Set ID sent in pulses
emit_pulse(void)Broadcast + multicast + LAN sweep
emit_listen(...)select then drain
emit_recv_drain(...)Non-blocking drain only
emit_get_socket_fd(void)Fd for application select
emit_cleanup(void)Leave multicast, close socket

Callback type peer_callback_t receives peer_info_t (peer_id, ip_address, port, timestamp).

Troubleshooting

  • Both machines must be on the same LAN subnet for broadcasts to apply.
  • Disable AP / client isolation on the router if wireless clients cannot see each other.
  • Allow UDP port 9999 inbound on host firewalls if discovery is one-way.
  • VPN tunnels may capture the default route and interfere with LAN discovery.
